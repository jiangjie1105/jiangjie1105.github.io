内存管理

  • 内存： 由可读写单元组成，表示一片可操作空间
  • 管理：人为的去操作一片空间的申请，使用和释放
  • 内存管理：开发者主动申请空间，使用空间，释放空间
  • 管理流程： 申请-使用-释放

JavaScript中的内存管理

  • 申请内存空间
  • 使用内存空间
  • 释放内存空间

JavaScript中的垃圾回收

  • JavaScript中内存管理是自动的
  • 对象不再引用时时垃圾
  • 对象不能从根上访问时是垃圾

JavaScript中的可达对象

  • 可以访问到的对象就是可达对象
  • 可达的标准就是从根出发是否能够被找到
  • JavaScript中的根就可以理解为是全局变量对象

GC算法

GC定义与作用

  • GC就是垃圾回收机制的简写
  • GC可以找到内存中的垃圾，并释放和回收空间
    GC里的垃圾是什么：

GC算法是什么

  • GC是一种机制，垃圾回收器完成具体的工作
  • 工作的内容就是查找垃圾释放空间，回收空间
  • 算法就是工作时查找和回收所遵循的规则

常见GC算法

  • 引用计数
  • 标记清除
  • 标记整理
  • 分代回收

5.引用计数算法

  • 核心思想：设置引用数，判断当前引用数是否为0
  • 存在引用计数器
  • 引用关系发生改变时引用计数器修改引用数字
  • 引用数字为0时立即回收

引用计数优点

  • 1：发现垃圾时立即回收
  • 2：最大限度减少程序暂停

缺点

  • 1.无法回收循环引用的对象
  • 2.时间开销大

标记清楚算法实现原理

  • 核心思想： 将垃圾回收过程分为两个阶段：1.标记 2.清除
  • 1.遍历所有对象找标记活动对象
  • 2.遍历所有对象清除没有标记对象
  • 经过两次遍历行为回收相应的空间
    把回收的空间放到空闲列表 以备使用

标记清除算法优缺点

对比引用计数的优点： 可以解决对象循环引用的回收操作，如果定义了函数互相引用，那么全局global对象就无法访问到函数内部的对象 那么这两个互相引用的函数就变成了“不可达对象” 对于不可达对象，在第一次标记时 就无法标记 而在第二次清除时就会被垃圾回收装置回收

标记清除算法缺点 ：1.地址不连续 会存在空间碎片化的问题 如果我们回收的垃圾对象在地址上是不连续的，由于这个不连续 ，造成了它们被回收之后分散到了各个角落，后续想要使用时如果新的生成空间与他们的大小匹配，那么久刚好，如果不匹配 ，多了或者少了就不适合使用标记清楚算法
2. 它会递归遍历root,这样

标记整理算法原理

标记整理算法原理：

  • 标记整理可以看作是标记清除的增强
  • 标记阶段的操作和标记清除一致
  • 清除阶段会先执行整理，移动对象位置

会在之前基础上整理活动对象 整理出连续空间 方面后续使用

常见GC算法总结

  • 引用技术 内部有引用计数器
    • 优点：
      • 可以及时回收垃圾对象
      • 减少程序卡顿时间
    • 缺点：
      • 无法回收循环引用的对象
      • 资源消耗较大
  • 标记清除：1遍历所有对象，标记活动对象2.清除未标记的对象
    • 优点：
      • 可以回收循环引用的对象
    • 缺点：
      • 容易产生碎片化空间，浪费空间
      • 不会立即回收垃圾对象
  • 标记整理
    • 优点：
      • 减少碎片化空间
    • 缺点：
      • 不会立即回收垃圾对象

认识V8

  • v8是一款主流的JavaScript执行引擎
  • v8采用即时编译
  • v8内存设限

V8垃圾回收策略

  • 采用分代回收的思想
  • 内存分为新生代，老生代
  • 针对不同对象采用不同算法

V8中常用GC算法

  • 分代回收（因为v8内存是有上限的）
  • 控件复制
  • 标记清除
  • 标记整理
  • 标记增量

V8如何回收新生代对象

V8内存分配

  • v8内存空间一分为二
  • 小空间用于存储新生代对象
  • 新生代指的是存活时间较短的对象（如局部作用域中的变量）

新生代对象回收实现

  • 回收过程采用复制算法 + 标记整理
  • 新生代内存区分为两个等大小空间
  • 使用空间为from，空闲空间为to
  • 活动对象存储与from空间
  • 标记整理后将活动对象拷贝至to
  • from 与 to 交换空间完成释放

回收细节说明

  • 拷贝过程中可能出现晋升
  • 晋升就是将新生代对象移动至老生代
  • 一轮GC还存活的新生代需要晋升
  • To空间的使用率超过25%

V8如何回收老生对象

老生代对象说明

  • 老生代对象放在右侧老生代区域
  • 64位操作系统1.4G，32位操作系统700M
  • 老年代对象就是指存活时间较长的对象

老生代对象回收实现

  • 主要采用标记清除，标记整理，增量标记算法
  • 首先采用标记清除完成垃圾空间的回收
  • 采用标记整理进行空间优化
  • 采用增量标记进行效率优化

细节对比

  • 新生代区域垃圾回收使用空间换时间
  • 老生代区域垃圾回收不适合复制算法（1.存放对象数据比较多 复制过程耗时，所以不合适 2. 一分为二浪费空间）

标记增量如何优化垃圾回收

程序和垃圾回收装置交替执行
程序执行==> 遍历对象进行标记（这一便标记直接可达的对象） ==> 执行 ==> 标记（标记间接可达的对象） ==> 完成清除 ==> 执行

V8垃圾回收总结

  • V8是一款主流的JavaScript执行引擎
  • V8内存设置上限（1.它本身为浏览器设置的，对于web应用 这么大足够了 2.由内部垃圾回收机制决定，如果内存大一些，回收时间可能会超过用户感知。）
