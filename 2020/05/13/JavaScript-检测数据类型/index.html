JavaScript-检测数据类型

2020-05-13
Word count: 1.1k   |   Reading time≈ 4 min

JavaScript中常常要用到数据类型的判断，这是一个很重要的问题，今天在这里总结以下我所理解的js数据类型判断的几种方式

1.typeof关键字

console.log(typeof 2);               // number
console.log(typeof true);            // boolean
console.log(typeof 'str');           // string
console.log(typeof []);              // object     []数组的数据类型在 typeof 中被解释为 object
console.log(typeof function(){});    // function
console.log(typeof {});              // object
console.log(typeof undefined);       // undefined
console.log(typeof null);            // object     null 的数据类型被typeof 解释为 object

可以看到，typeof几乎可以正确的检测大多数数据类型，但是空数组和null被typeof解释为object类型，这种说法不能说不对，只能说不够精确而已，那么要如何解决这个问题，就要用到下面这种方法。

2.instanceof关键字


console.log(2 instanceof Number);                 // false
console.log(true instanceof Boolean);           // false 
console.log('str' instanceof String);           // false  
console.log([] instanceof Array);                  // true
console.log(function(){} instanceof Function);     // true
console.log({} instanceof Object);                 //true

以上结果显示，只有array，object，function被精确判断，其他（number，boolean，string）都不能被准确判断其实instanceof在MDN中这样解释：instanceof运算符用来测试一个对象在其原型链中是否存在一个构造函数的prototype属性，意思就是判断对象是否是某一数据类型（如Array）的实例，重点关注以下判断一个对象是否是数据类型的实例，在这里字面量值 2，true ，‘str’不是实例，所以值是flase

尤其是null，其实这是js发展过程中的重大失误，早期准备更改null的类型为null，由于当时已经由大量网站使用了null，如果更改，将导致很多网站的逻辑出现漏洞问题，就没有更改过来，于是一直遗留到现在。

3.constructor

console.log((2).constructor === Number);
console.log((true).constructor === Boolean);
console.log(('str').constructor === String);
console.log(([]).constructor === Array);
console.log((function() {}).constructor === Function);
console.log(({}).constructor === Object);

可以看到，用constructor来判断类型看起来比较完美，然而，如果创建一个对象，更改它的原型，这种方法也变得不可靠了。

function Fn(){}
Fn.prototype = new Array();
var f = new Fn();

console.log(f.constructor===Fn);//false
console.log(f.constructor===Array);//true

如此轻易的就更改了constructor，到这里，我们应该需要一个新的大杀器，最精准的判断数据类型，那么可以用下面的方法

Object.prototype.toString.call()

var a = Object.prototype.toString

console.log(a.call(1));
console.log(a.call(true));
console.log(a.call('str'));
console.log(a.call([]));
console.log(a.call(function(){}));
console.log(a.call({}));
console.log(a.call(undefined));
console.log(a.call(null));

我们使用的是Object对象的原型方法toString，使用call狸猫换太子，接用Object的toString方法

结果精确的显示了我们需要的数据类型。

总结：

上述我们提到了四种方法检测数据类型，其中：

1.typeof：

typeof对于：number，string，undefined，boolean，symbol，function，对象object 都有精准的解释，但是对数组和null解释不精确
优点：能快速检查undefined,string,number,boolean类型

缺点：当类型为object,null,array时都会返回object,所以不能区分这三类

2.instanceof

instanceof基于原型链的查找，它是一个操作符，判断是一个对象是否是另一个对象的实例。返回true或false。他对于引用数据类型（object function array）可以精确判断，但是对于基本数据类型不能精准判断 因为它们不是一个数据类型的实例
优点：能检测array,function,object类型

缺点:检测不了number,boolean,string

3.constructor

constructor看起来比较完美，但是更改了对象的原型就无法判断了

4.Object.prototyp.toString.call()

优点：能准确的判断所有的类型。

缺点：写法过于繁琐
